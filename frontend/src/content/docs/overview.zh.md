# 概览（Overview）

Sub2API 是一个 **自托管的 AI API Gateway**：把多种上游（订阅/OAuth/API Key）能力收敛成统一的 API 入口，并提供配额分发、API Key 管理、计费与风控能力。

## 适用人群

- 想把“订阅额度/多个上游账号”稳定分发给团队成员或系统调用方
- 需要统一的 API Key、分组、限流、并发控制、用量统计与审计
- 希望在不暴露上游凭据的前提下，提供标准化的 API 访问

## 核心概念

- **上游账号（Account）**：管理员在后台录入/授权的上游身份（OAuth / API Key 等）。
- **下游 API Key（User Key）**：给调用方使用的 Key（平台签发），调用时只需要携带它。
- **网关接口（Gateway API）**：统一入口（例如 `/v1/messages`、`/v1/chat/completions`、`/v1beta/models/*`）。
- **调度与风控**：在多个上游账号间做选择与限流（目标是更稳定、更可控地消耗额度）。

## 安全边界（重要）

为了防止通过逆向或接口探测拿到上游信息，Sub2API 的设计原则是：

- **最小暴露**：普通 API/页面不会返回上游凭据明文，也不会暴露内部链路/映射细节。
- **管理端受控**：只有管理员在受控流程下才能查看/操作敏感内容（且建议开启审计与最小权限）。
- **默认不收录**：自建实例默认处于 “private noindex” 索引模式，避免被搜索引擎/AI 搜索误收录。

> 提示：当你们后续确定了官方域名并准备做 SEO，再把索引模式切到 public。
