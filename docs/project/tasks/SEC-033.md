## SEC-033 — Security/DoS: cap remote HTTP response bodies (pricing/hash/checksum)

### Problem
Several remote HTTP fetch paths read response bodies with `io.ReadAll(resp.Body)` without a size cap. A misconfigured or malicious remote endpoint can return an arbitrarily large body and cause excessive memory usage (DoS).

### Evidence (code)
- Remote pricing JSON fetch reads unbounded: `backend/internal/repository/pricing_service.go:50`
- Remote hash text fetch reads unbounded: `backend/internal/repository/pricing_service.go:69`
- GitHub checksum fetch reads unbounded: `backend/internal/repository/github_release_service.go:140`

### Goal
Add explicit maximum response size limits for the above remote fetches, returning a deterministic error when exceeded.

### Acceptance Criteria
- [ ] `FetchPricingJSON` enforces a max body size (default: 5 MiB).
- [ ] `FetchHashText` enforces a max body size (default: 64 KiB).
- [ ] `FetchChecksumFile` enforces a max body size (default: 256 KiB).
- [ ] New unit tests cover the “too large response body” case.
- [ ] Verification: `cd backend; go test ./...` PASS.
