## SEC-042 — Security/SSRF: default `allow_private_hosts` to false

### Problem
`security.url_allowlist.allow_private_hosts` currently defaults to `true`. This weakens SSRF protections because it allows upstream URLs that resolve to private/loopback/link-local IP ranges (depending on call sites), which can be abused for internal network probing or metadata service access if an attacker can control `base_url` or otherwise influence upstream targets.

Given this repo is a gateway that forwards user requests to upstreams, secure defaults should reject private hosts unless the operator explicitly opts in.

### Evidence (code)
- Default allows private hosts: `backend/internal/config/config.go:914`
- URL validation uses `AllowPrivate`/`AllowPrivateHosts`:
  - `backend/internal/service/openai_gateway_service.go:1923` (`AllowPrivate: s.cfg.Security.URLAllowlist.AllowPrivateHosts`)
  - `backend/internal/service/gateway_service.go:4373`
  - `backend/internal/service/pricing_service.go:441` (pricing host fetches)
  - `backend/internal/pkg/httpclient/pool.go:86` (ValidateResolvedIP gating)

### Goal
Make private/loopback hosts opt-in by defaulting `security.url_allowlist.allow_private_hosts` to `false`.
