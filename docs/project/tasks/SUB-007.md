# SUB-007: User: subscription single progress endpoint + contract alignment

> Status: ✅ Verified (2026-01-30)
> 验证：`cd backend; go test ./...`、`pnpm -C frontend test:run`、`pnpm -C frontend build`

## 背景/问题

前端需要按订阅 ID 查询单条订阅进度（用于用户侧详情/卡片展示），契约为：

- `GET /api/v1/subscriptions/:id/progress`

## 现状（已实现）

- 前端封装：`frontend/src/api/subscriptions.ts#getSubscriptionProgress`
- 后端路由：`backend/internal/server/routes/user.go` 已包含：
  - `subscriptions.GET("/:id/progress", h.Subscription.GetProgressByID)`

## 验收标准

- `GET /api/v1/subscriptions/:id/progress` 在用户 JWT 鉴权下可用
- `:id` 为路径参数，返回结构与前端 `SubscriptionProgress` 类型一致
- 未登录/无权限返回 401/403（按现有中间件策略）
