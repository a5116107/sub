# SEC-031: Security/DoS: proxy probe should avoid plaintext HTTP and cap response body

> Status: ✅ Completed (2026-02-03)
> 验证：`cd backend; go test ./...`

## 背景/问题（以代码为准）

代理连通性检测（Proxy exit info probe）使用固定第三方接口查询出口 IP 信息：

- 默认 `ipInfoURL` 使用明文 HTTP（存在中间人篡改风险）
- 读取响应体时未设置上限（第三方/代理异常时可能导致内存占用异常）

证据：
- `backend/internal/repository/proxy_probe_service.go:39`（默认 URL）
- `backend/internal/repository/proxy_probe_service.go:92`（读取响应体）

## 修复

- 默认 URL 改为 HTTPS：`https://ip-api.com/json/?lang=zh-CN`
- 读取响应体增加上限：`maxProxyProbeBodyBytes = 1<<20`

## 验收标准

- [x] 默认不再使用明文 HTTP
- [x] probe 读取响应体有明确上限
- [x] `cd backend; go test ./...` PASS
