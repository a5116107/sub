# SEC-029: Privacy: redact OAuth exchange logs (email/org/account uuid)

> Status: ✅ Verified (2026-02-02)
> 验证：`cd backend; go test ./...` PASS

## 背景/问题（以代码为准）

OAuth 换 token 时打印了敏感字段到日志：

- 证据：`backend/internal/service/oauth_service.go`
  - `log.Printf("[OAuth] Got org_uuid: %s", ...)`
  - `log.Printf("[OAuth] Got account_uuid: %s", ...)`
  - `log.Printf("[OAuth] Got email_address: %s", ...)`

影响：
- 日志侧泄露 email / account UUID / org UUID（可被集中日志、Sentry、运维输出等二次扩散）。

## 目标

- 默认不在日志输出敏感字段明文；必要时仅输出脱敏后的摘要（便于排障）。

## 方案

- 将上述日志改为：
  - 不打印 email；
  - UUID 仅打印掩码（例如前 6 + 后 4），或仅打印 “present=true”。

## 验收标准

- [ ] 不再出现明文 email/org_uuid/account_uuid 输出
- [ ] `cd backend; go test ./...` PASS
