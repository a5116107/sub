## SEC-039 — Privacy: eliminate fmt.Printf logging in OAuth flows (use structured logger + redaction)

### Problem
Several OAuth-related code paths log via `fmt.Printf`, which bypasses the project's structured logging/redaction approach and can leak sensitive data (emails, proxy URLs, tier IDs, etc.) to stdout/stderr.

### Evidence (code)
- Antigravity OAuth prints errors and token refresh details: `backend/internal/service/antigravity_oauth_service.go:140`, `backend/internal/service/antigravity_oauth_service.go:148`, `backend/internal/service/antigravity_oauth_service.go:175`
- Antigravity token refresher prints account and expiry details: `backend/internal/service/antigravity_token_refresher.go:46`
- Gemini OAuth service prints warnings and tier details: `backend/internal/service/gemini_oauth_service.go:544`, `backend/internal/service/gemini_oauth_service.go:554`, `backend/internal/service/gemini_oauth_service.go:601`, `backend/internal/service/gemini_oauth_service.go:622`, `backend/internal/service/gemini_oauth_service.go:816`, `backend/internal/service/gemini_oauth_service.go:901`, `backend/internal/service/gemini_oauth_service.go:903`
- Drive client prints API errors via stdout: `backend/internal/pkg/geminicli/drive_client.go:121`
- Code Assist repo client prints request/response data via stdout: `backend/internal/repository/geminicli_codeassist_client.go:37`, `backend/internal/repository/geminicli_codeassist_client.go:54`, `backend/internal/repository/geminicli_codeassist_client.go:45`

### Goal
Replace `fmt.Printf` usages in these runtime code paths with the project's logger (`log.Printf` already used in the same services) and ensure sensitive fields are not logged.

### Acceptance Criteria
- [ ] Remove/replace `fmt.Printf` logging in runtime OAuth services (Gemini/Antigravity).
- [ ] Do not log tokens, auth codes, refresh tokens, or secrets.
- [ ] Logs remain useful for debugging (context + error), using a consistent prefix and severity.
- [ ] Verification: `cd backend; go test ./...` PASS.
