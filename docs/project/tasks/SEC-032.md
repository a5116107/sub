# SEC-032: Security/DoS: opencode instructions fetch must have timeout and size limit

> Status: ✅ Completed (2026-02-03)
> 验证：`cd backend; go test ./...`；`pnpm -C frontend test:run`；`pnpm -C frontend build`

## 背景/问题（以代码为准）

OpenAI 网关在非 Codex CLI 请求、且请求体未提供 `instructions` 时，会调用 `GetOpenCodeInstructions()` 作为默认指令。
该函数在缓存失效/缺失时会去 GitHub Raw 拉取 `opencode` 的 `codex_header.txt`。

问题：

- 网络请求使用 `http.DefaultClient`，无超时控制；在网络抖动/阻断时会把请求处理线程拖住（可被高频请求放大为 DoS）。
- `io.ReadAll(resp.Body)` 无响应体上限；理论上可被异常响应放大为内存风险。

证据：
- `backend/internal/handler/openai_gateway_handler.go:128`（触发路径：读取默认 instructions）
- `backend/internal/service/openai_codex_transform.go:16`（固定 URL）
- `backend/internal/service/openai_codex_transform.go:525`（原 fetch 逻辑）

## 修复

- 增加超时与上限：
  - `opencodeFetchTimeout = 3s`
  - `opencodeFetchMaxBytes = 256KB`
- 拉取逻辑使用 `context.WithTimeout` + `http.Client{Timeout: ...}` 并用 `io.LimitReader` 读取。
- 新增单测覆盖“超大响应体会被拒绝”。
