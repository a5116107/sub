## SEC-041 — Security: default `allow_insecure_http` to false (prevent plaintext upstreams)

### Problem
`security.url_allowlist.allow_insecure_http` currently defaults to `true`. When URL allowlist checks are disabled (or in any code path that falls back to `ValidateURLFormat`), this allows user-configured upstream `base_url` to be plain HTTP. That enables MITM, credential/token leakage, and downgrade attacks. Even if allowlist is typically enabled, this is a risky default and can be reintroduced by operator misconfiguration.

### Evidence (code)
- Default allows insecure HTTP: `backend/internal/config/config.go:915`
- Services use `ValidateURLFormat(..., AllowInsecureHTTP)` when allowlist is disabled:
  - `backend/internal/service/openai_gateway_service.go:1917`
  - `backend/internal/service/gateway_service.go:4367`
  - `backend/internal/service/gemini_messages_compat_service.go:397`
  - `backend/internal/service/pricing_service.go:435`

### Goal
Make plaintext HTTP upstreams opt-in by defaulting `security.url_allowlist.allow_insecure_http` to `false`.
